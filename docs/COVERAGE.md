"Implemented" means the name resolves and behaves; "oracle-tested"
means at least one `Oracle/` case exercises it and matches Excel. The
gap between them is mostly the connector functions that need a backend
to test against, plus a handful of clock-sensitive and
compile-rejected functions (below).

## What's fully covered

The data-shaping core — everything you'd actually write a query with:

- **`Table.*`** (114), **`List.*`** (72), **`Record.*`** (18) — the
  tabular trio.
- **`Text.*`**, **`Number.*`**, **`Binary.*`**, **`Logical.*`** — scalars,
  including the corner cases the oracle caught (`Single.From` f32
  round-trip, `Binary.From` base64 decode, `BinaryFormat` big-endian
  defaults, 7-bit signed zigzag).
- **`Date.*`**, **`DateTime.*`**, **`DateTimeZone.*`**, **`Duration.*`**,
  **`Time.*`** — temporal, with .NET-tick fractional-second precision in
  JSON output.
- **`Value.*`**, **`Type.*`** and the whole `*.Type` token set — the type
  system, including the discovery that enum-family `.Type` constants are
  the *type-of-types*, not numeric aliases.
- **`Json.*`**, **`Csv.*`**, **`Xml.*`**, **`Html.*`**, **`Lines.*`** —
  document parse/serialise.
- **`Geography.*`**, **`Geometry.*`** and their `*Point.From` constructors
  — WKT `POINT` round-trips.
- **`RowExpression.*`**, **`ItemExpression.*`** — lambda-body AST
  reflection (useful for introspecting a filter predicate without
  re-parsing source).
- **`Splitter.*`**, **`Combiner.*`**, **`Comparer.*`**, **`Replacer.*`**,
  **`Uri.*`**, **`Function.*`**, **`Expression.*`**, **`Diagnostics.*`**,
  **`Variable.*`**.

## Implemented, not oracle-testable

These resolve and run, but can't be pinned by a byte-diff:

- **Clock-tight predicates** — `DateTime.IsInCurrentSecond`,
  `IsInNext/PreviousMinute`, `IsInNext/PreviousSecond`. Excel and
  mrsflow must agree on "now" to the second; they race.
- **Compile-rejected-by-Excel** — `Value.Firewall`, `Value.NativeQuery`,
  `Value.VersionIdentity`, `Function.ScalarVector`. Excel refuses to
  evaluate them outside a real connector/firewall context, so there's
  no oracle output to match.
- **API-shape mismatches** — `Table.MaxN`/`MinN` (Excel wants a lambda
  where mrsflow takes a column name), `Table.FromPartitions`,
  `Function.InvokeWithErrorContext` (error-wording differs in a way
  that's not worth a codebase-wide change).
- **Page-reader infra** — `List.ConformToPageReader`,
  `Table.ConformToPageReader`. Power BI dataflow paging machinery with
  no off-cloud meaning.
- **Handler dispatch** — `Table.View` honours a `GetRowCount` handler in
  Excel; mrsflow returns the real count. A faithful implementation needs
  view-handler plumbing through every `Table.*` reader.
- **Column-type metadata** — `Table.AddFuzzyClusterColumn`, full
  `Table.AddKey`/`Keys` round-tripping. mrsflow doesn't track per-column
  type or key metadata on the table value yet.

## Deliberately unimplemented (the 80 PQ-only names)

Almost all are **cloud connectors that need a backend mrsflow doesn't
have**. Implementing them means a network/auth integration, not an M
semantics question, so they're out of scope for a local data tool:

| Bucket                    | Families                                                              |
| ------------------------- | -------------------------------------------------------------------- |
| Cube / OLAP (16)          | `Cube.*` — the largest single family, all SSAS cube operations       |
| Azure                     | `AzureStorage.*` (5), `AzureDataExplorer.*` (3), `DataLake.*`, `Kusto.*`, `Fabric.*`, `Lakehouse.*` |
| Microsoft 365             | `SharePoint.*` (3), `Exchange.*`, `Graph.*`, `PowerPlatform.*`, `CommonDataService.*`, `Office.*` |
| Hadoop ecosystem          | `HdInsight.*` (3), `Hdfs.*` (2)                                       |
| RDBMS connectors          | `OleDb.*` (2), `AdoDotNet.*` (2), `Oracle.*`, `DB2.*`, `Sybase.*`, `Teradata.*`, `Informix.*`, `Progress.*`, `SapHana.*` |
| Other SaaS                | `Salesforce.*` (2), `AdobeAnalytics.*`, `Soda.*`, `Snowflake.*`       |
| Connector internals       | `AccessControlEntry.*` (2), `Identity.*` (2), `Resource.*`, `SqlExpression.*` (2), `DirectQueryCapabilities.*`, `Embedded.*`, `Tables.GetRelationships`, `RData.FromBinary`, `ActiveDirectory.*`, `OData.Feed` |
| Deprecated                | `Excel.RichDocument` (PowerView format)                              |

A few non-connector PQ-only names are also skipped because Excel itself
rejects them in a plain expression context (they only work inside a
connector's fold callback): `Value.ResourceExpression`,
`IdentityProvider.Default`, `SqlExpression.SchemaFrom`/`.ToExpression`.

## mrsflow-only extensions

Four names exist in mrsflow's `#shared` but not PQ's — these are
implementation helpers (e.g. the `EvalFile` workbook wrapper used by the
test harness) and a couple of native-driver query entry points
(`MySQL.Query`, `PostgreSQL.Query`) that mirror `Odbc.Query`.
